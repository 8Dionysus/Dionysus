## Contract being planted

This planting plan proposes the first honest live wave for the
`aoa-stats-telemetry` pack.
It does not treat telemetry as one sovereign new owner.
Instead it stages one split wave led by `aoa-skills` and `aoa-evals`, where
owner-local receipts become real first and any federated summaries remain
deferred.

The contract being made explicit is:

- machine-readable receipts should stay close to the owner surface that emits
  them
- proof meaning should remain in `aoa-evals`
- workflow meaning should remain in `aoa-skills`
- later read models should derive from those surfaces rather than replace them

## Deferred zones / non-goals

- no sovereign telemetry repo from this first wave
- no cross-repo read model in the first landing
- no replacement of verdicts with counts
- no hidden score for route quality or quest progression
- no first-authoring into `aoa-routing` or `aoa-kag`
- no automatic writeback into `aoa-memo` or checkpoint law into `aoa-agents`
  before receipts stabilize

## Risks / follow-ups

- if `aoa-skills` receipts grow too large, the layer will start duplicating raw
  artifacts instead of linking to them
- if `aoa-evals` receipts drift into interpretation changes, telemetry will
  start masquerading as proof authority
- if the envelope fields diverge between repos, a later derived layer will be
  brittle from day one
- if the next step becomes "dashboard first", the project will likely create a
  stats shell before it has stable facts to derive from
- after slices 1 to 3 land, reassess whether the next honest owner is a new
  derived stats adjunct or a smaller follow-on inside existing repos
