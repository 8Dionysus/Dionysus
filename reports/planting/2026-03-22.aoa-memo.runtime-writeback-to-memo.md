## Contract being planted

This planting makes the runtime-to-memo writeback seam explicit without turning `aoa-memo` into a runtime store or hidden orchestration layer.
It adds one bounded seam note, one schema-backed checkpoint-to-memory contract, one example mapping surface, and discoverability updates in the generated memo doctrine routes.
The current object canon stays intact: the planting clarifies how runtime-local state may map into existing memo kinds rather than inventing a new memory-object family.
