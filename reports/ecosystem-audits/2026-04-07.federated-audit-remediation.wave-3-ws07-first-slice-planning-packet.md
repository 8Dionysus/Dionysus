# Federated Audit Remediation Wave 3 WS07 First-Slice Planning Packet

Date: `2026-04-07`

## Goal

Land the narrowest owner-first `WS07` slice: expose compact upstream
proof-lineage in `aoa-evals` reader surfaces and split maturity from
default-reference readiness in `aoa-skills` public governance surfaces without
widening bundle claims, governance backlog doctrine, or Wave 3 docs scope.

## Repositories

- `aoa-skills`
- `aoa-evals`

## Non-goals

- no new skill promotions or governance-lane decisions
- no redesign of `generated/governance_backlog.*`
- no broader release/onboarding/support rewrite from `WS11`
- no bundle-local claim changes in `aoa-evals`
- no new comparison doctrine, rubric semantics, or scorer authority
- no repo-wide runtime/export redesign beyond the release-manifest refresh

## Validator Of Record

Per repo:

- `aoa-skills`: `python scripts/validate_skills.py --fail-on-review-truth-sync`
- `aoa-evals`: `python scripts/validate_repo.py`

## Stop Condition

Stop this slice once:

- `aoa-skills` public surface exposes source-owned maturity separately from
  derived default-reference readiness
- project overlays without a governance lane read as `not_applicable` instead
  of looking silently blocked
- `aoa-evals` thin reader surfaces expose compact `skill_refs`,
  `technique_refs`, `evidence_kinds`, and `proof_surface_kinds`
- `aoa-evals` capsules stay compact while carrying the same bounded lineage
  signals as the min catalog

## Rollback Marker

If this slice requires rewriting governance backlog semantics, changing
bundle-local proof claims, or widening into Wave 3 onboarding/release docs,
stop and roll back to the last state where both repo-native validator stacks
still pass with the narrower public-surface and reader-surface contract only.
