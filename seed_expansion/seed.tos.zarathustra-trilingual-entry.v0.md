## Language Role Contract

### Core move

Fix asymmetric language roles before the first entry spreads meaning across three languages.

### Why now

- three parallel language trees would drift faster than ToS can honestly review them
- source-first authority requires a stronger source-language anchor
- AoA, KAG, and public node IDs already lean on English structural surfaces, while lived commentary can remain richer in Russian

### First materialized output

- German as canonical source authority
- Russian as human-reviewed commentary and working translation layer
- English as public structural, routing, and interoperability layer
- explicit note that concepts remain language-neutral and are not identical to any one lexeme

## Single-Tree Witness Discipline

### Core move

Grow one `node_id` with multiple language witnesses, not three node copies.

### Why now

- ToS already keeps source, interpretation, and derived layers visibly distinct
- translations should remain witness-linked and reviewable
- concept nodes need multilingual labels without becoming flat synonym piles

### First materialized output

- paragraph- or sense-block alignment between `de / ru / en` witnesses
- witness roles such as `canonical_source`, `working_translation`, and `bridge_translation`
- translation-tension notes only where philosophically load-bearing
- first rule that the German source may be challenged by commentary, but not replaced by it
