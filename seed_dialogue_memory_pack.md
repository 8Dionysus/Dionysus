# Seed Note: Dialogue Memory Pack

## Purpose

Prepare the dialogue-derived memory pack from
`dionysus_dialogue_seed_pack_2026-03-24.zip` as a bounded future prep surface
inside `Dionysus`.

This pack preserves three future memory-adjacent seeds without pretending they
already own registry state, wave order, or current planting priority.

## Why this belongs in Dionysus

`Dionysus` is the seed garden and dispatch layer.
This prep pack belongs here because:

- the three seeds cross several owning repositories
- they are future-facing and should not displace the live ToS gate
- the included registry append is only conditional, not ready to merge unchanged
- the pack works best as a later bounded planting source, not as immediate repo ownership

## Bundle contents to preserve

The source artifact is `dionysus_dialogue_seed_pack_2026-03-24.zip`.

It carries:

- `README.md`
- `seed_expansion/seed.8dionysus.aoa.derived-memory-dual-substrate.v0.md`

## Ownership posture

Read later landings under these repo-home boundaries:

- `aoa-memo` owns explicit memory objects and writeback discipline
- `aoa-kag` owns typed derived lifts and graph-facing substrate surfaces
- `ATM10-Agent` owns hybrid retrieval posture and runtime memory profiles
- `aoa-evals` owns fidelity, fallback, and promotion proof
- `aoa-agents` owns hot-memory posture where runtime control needs role/tier linkage
- `abyss-stack` owns substrate-side runtime memory profile surfaces
- `Tree-of-Sophia` owns source-authored meaning and only receives bounded export-boundary notes

This pack stages later memory plantings for those owners.
It does not move their truth into `Dionysus`.

## Pack contents to preserve

The bounded seed set is:

1. `Derived Memory Dual-Substrate`
2. `Typed Provenance Lifts`
3. `DiffMem Hot-Memory Controller`

## Suggested planting order

Keep the source order from the bundle:

1. Derived Memory Dual-Substrate
2. Typed Provenance Lifts
3. DiffMem Hot-Memory Controller

That order keeps:

- explicit memory before graph ambition
- provenance before lift inflation
- runtime augmentation after deterministic contracts

## What this pack is not for

- no change to `navigation.next_live_seed`
- no `tenth_wave.manifest.json`
- no silent `seed_index` promotion
- no graph empire in place of source-owned truth
- no differentiable memory as a new canonical store
- no quiet collapse between `aoa-memo`, `aoa-kag`, and `Tree-of-Sophia`

## Success sign

This pack is planted well when:

- authored truth remains upstream
- explicit memory remains inspectable
- graph and vector planes stay downstream and provenance-rich
- runtime additions degrade honestly
- eval gates remain stronger than diagram ambition

## Final rule

Use this prep pack to hold the dialogue-derived memory seeds until one later
planting opens them one at a time.
Do not let an attractive memory idea skip the source-first and proof-first boundaries already present in the ecosystem.
