# Seed Note: Questbook Foundation Pack

## Purpose

Prepare the questbook foundation rollout from
`archive/seed_pack_exports/questbook_first_wave_seed.zip` as a named prep pack
inside `Dionysus` without opening a tenth numbered wave.

This pack stages the shared questbook model, delegation passport, routing seam,
playbook harvest boundary, and memo writeback boundary before questbook spreads
further across the ecosystem.

## Why this belongs in Dionysus

`Dionysus` is the seed garden and dispatch layer.
This prep pack belongs here because:

- the rollout spans multiple owning repositories
- the current next-live seed should remain the ToS expansion gate
- the work needs priority and dependency control without pretending to be a new numbered wave
- `ATM10-Agent` is now out of scope for this planting line and should not be used as the proving ground here

## Bundle contents to preserve

The source artifact is `archive/seed_pack_exports/questbook_first_wave_seed.zip`.

It carries:

- `questbook_first_wave_seed/README.md`
- `questbook_first_wave_seed/APPLY_ORDER.md`
- `questbook_first_wave_seed/SEED_MANIFEST.yaml`
- `questbook_first_wave_seed/TOUCHPOINTS_NOT_INCLUDED.md`
- `questbook_first_wave_seed/later-wave-targets.md`
- repo-local questbook starter surfaces for `Agents-of-Abyss`, `aoa-agents`,
  `aoa-routing`, `aoa-playbooks`, and `aoa-memo`

## Why this is packed instead of registered now

The current `seed_index` contract is still wave-shaped:

- `wave: null` entries are reserved for the single live next seed or planted post-wave material
- this questbook work is neither of those
- the honest form is a named prep pack with explicit priority and dependencies

So the clean path is:

1. keep the current live seed intact
2. keep questbook rollout as a named prep pack
3. plant the smallest bounded slice when a real owner repo is ready
4. only open a numbered wave later if the rollout truly becomes a bounded wave campaign

## Ownership posture

Read the later plantings under these repo-home boundaries:

- `Agents-of-Abyss` owns the common questbook model and first shared boundary notes
- `aoa-agents` owns the execution passport and delegation posture
- `aoa-routing` owns thin derived discovery, not quest meaning
- `aoa-playbooks` owns harvest and reanchor semantics
- `aoa-memo` owns recall and writeback boundaries

This pack stages the shared foundation.
It does not turn `Dionysus` into the owner of quest meaning.

## Scope exclusions

This pack explicitly excludes:

- `ATM10-Agent`, which is tracked separately on Windows
- `aoa-sdk`, which should become a later separate named seed after the current questbook contour is fully planted
- second-line source/proof, boundary/runtime, and profile/seed-garden rollout work

## Suggested planting order

Keep the default order flexible but bias toward:

1. `Agents-of-Abyss`
2. `aoa-agents`
3. `aoa-routing`
4. `aoa-playbooks`
5. `aoa-memo`

## Final rule

Land the foundation before widening the questbook contour.
Do not let early questbook rollout skip the shared ownership and boundary seams
that make later repo-local adoption honest.
