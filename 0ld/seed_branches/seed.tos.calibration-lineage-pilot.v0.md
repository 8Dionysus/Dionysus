Roadmap-derived seed source for the first bounded ToS lineage-branch pilot.

Date: 2026-03-22
Status: curated roadmap seed, not direct implementation backlog

## Grounding

The first ToS scaffold already exists:

- stable node IDs
- source-node template
- concept-node template
- one worked example pair

The next move should prove that scaffold can support one real small branch.
It should not yet become wider world-thought expansion.

---

<a id="tos-branch-lineage-node-template"></a>
## Lineage-Node Template

### Core move

Give ToS a first public lineage-node template that can tie a small route together without exploding into graph sprawl.

### Why now

- lineage already matters more than archive in ToS law
- the scaffold now needs a branch-facing form, not just source and concept forms
- one lineage template is enough to open the pilot without claiming a full branch ontology

### First materialized output

- lineage-node template note
- one lineage-node example in the pilot set

## Calibration-Family Branch Pilot

### Core move

Use the current calibration-family scaffold as one bounded pilot route.

### Why now

- the current example set already points toward a small calibration-family lineage
- this keeps the wave coherent without pretending the pilot family is universal
- it proves branch formation before cross-tradition or volume-heavy expansion

### First materialized output

- named branch-pilot note
- explicit route and limits

## Worked Pilot Node Set

### Core move

Extend the current example set into a real small branch by materializing the companion source node, companion concept nodes, and one lineage node.

### Why now

- the scaffold needs one real route, not just abstract templates
- a bounded node set is enough to prove branch legibility
- this remains smaller and safer than wider world-thought expansion

### First materialized output

- expanded example set for the pilot route
- manual review note confirming the pilot is real but bounded
