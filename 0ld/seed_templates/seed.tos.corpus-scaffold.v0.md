# seed.tos.corpus-scaffold.v0.md

Roadmap-derived seed source for the first ToS corpus scaffold wave.

Date: 2026-03-22
Status: curated roadmap seed, not direct implementation backlog

## Grounding

The seed-bundle era is now planted.

The next move should not reopen mixed bridge work or runtime instrumentation.
It should turn the new ToS growth law into the first public corpus scaffold:

- stable node identifiers
- first node templates
- one worked example set

This keeps ToS source-first while giving later corpus and branch growth a clearer body.

---

<a id="tos-template-stable-identifier-discipline"></a>
## Stable Identifier Discipline

### Core move

Give ToS a public `node_id` law:

`tos.<node_type>.<slug[.subslug...]>`

### Why now

- the node contract already exists, but `node_id` is still too loose
- later corpus growth needs stable handles before it needs more files
- downstream derived systems should meet authored identifiers, not invent them

### First materialized output

- identifier-discipline note in `Tree-of-Sophia`
- node schema tightened around the public `node_id` grammar

---

<a id="tos-template-source-node-template"></a>
## Source-Node Template

### Core move

Give ToS its first public source-node template as a bounded authored scaffold.

### Why now

- source nodes are the most natural first corpus surface
- template clarity protects later corpus growth from flat-note drift
- this can be done without opening a branch pilot or a heavier platform program

### First materialized output

- source-node template note
- one source-node example aligned with the existing node contract

---

<a id="tos-template-concept-node-template-worked-example"></a>
## Concept-Node Template Plus Worked Example

### Core move

Add the companion concept-node template and one small worked example set.

### Why now

- ToS needs to show how concept scaffolds differ from source scaffolds
- a worked example proves the identifier law and template law can coexist
- one bounded example set is enough for this wave; a branch pilot is not needed yet

### First materialized output

- concept-node template note
- one concept-node example that shares a family resemblance with the source example
