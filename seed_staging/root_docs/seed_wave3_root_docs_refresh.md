# Seed Note: Third-Wave Root Docs Refresh Pack

## Purpose

Prepare and preserve the third-wave root docs refresh bundle as one bounded
cross-repo source-linked pack inside `Dionysus`.

This pack keeps the root `README.md` and root `AGENTS.md` refresh legible as a
transport-and-lineage surface while the actual planted meaning remains in the
owning repositories.

## Why this belongs in Dionysus

`Dionysus` is the seed garden and dispatch layer.
This pack belongs here because:

- the refresh spans multiple owning repositories
- the bundle is stronger as transport and replay than as root-repo canon
- the current `navigation.next_live_seed` should remain untouched
- the changes are bounded to root routing and workflow surfaces rather than
  deeper doctrine rewrites

## Ownership posture

Read the planted root refresh under these repo-home boundaries:

- `aoa-techniques` owns reusable public practice canon
- `aoa-skills` owns bounded Codex-facing execution bundles
- `aoa-evals` owns bounded proof surfaces
- `aoa-agents` owns role and persona contracts
- `aoa-sdk` owns the local-first typed control plane
- `ATM10-Agent` owns the project-local operator companion surface

This pack tightens the roots for those owners.
It does not transfer their meaning into `Dionysus`.

## What this pack is for

Apply one bounded root-entry refresh that:

- makes root `README.md` surfaces shorter, faster to scan, and more route-first
- makes root `AGENTS.md` surfaces clearer about boundaries, workflow, and
  validation
- keeps repository names, ownership contours, and boundary language exact
- preserves repository-specific character while grouping root routes more cleanly

## What this pack is not for

- no deep doctrine rewrite
- no new wave manifest
- no `seed_index` promotion
- no replacement of `navigation.next_live_seed`
- no invented maturity, visibility, or roadmap claims
- no transfer of layer-owned meaning back into a root overview surface

## Recommended planting order

Keep the bundle order bounded:

1. `aoa-techniques`
2. `aoa-skills`
3. `aoa-evals`
4. `aoa-agents`
5. `aoa-sdk`
6. `ATM10-Agent`

This keeps canon and layer-owned meaning steady before the newest consumer and
project-facing companion surfaces are refreshed against them.

## Local workspace note

`ATM10-Agent` should be treated as a cloud-only target for this wave.
It may use a temporary working checkout for planting and merge, but it should
not be promoted here as a new local workspace home or topology exception.

## Final rule

Keep roots route-first and boundary-explicit.
Let linked docs carry deep doctrine, and let the owning repositories remain the
owners after the refresh lands.
