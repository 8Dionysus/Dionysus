# Seed Note: AoA Architecture-Fit Pack

## Purpose

Prepare the architecture-fit pack from `dionysus_architecture_fit_pack.zip` as a
bounded future prep surface inside `Dionysus` without treating it as a live
registry seed, a new wave, or a stealth `seed_expansion/` promotion.

This pack keeps the research donor about scaling agent systems legible as a set
of later bounded AoA plantings rather than letting one donor harden into
runtime canon by accident.

## Why this belongs in Dionysus

`Dionysus` is the seed garden and dispatch layer.
This prep pack belongs here because:

- it spans multiple owning repositories
- the current `navigation.next_live_seed` is already occupied by the ToS live gate
- the pack is stronger as staging than as an immediate `seed_index` promotion
- the repo needs one source-linked architecture-fit note before any downstream planting starts

## Bundle contents to preserve

The source artifact is `dionysus_architecture_fit_pack.zip`.

It carries:

- `dionysus_architecture_fit_pack/README.md`
- `dionysus_architecture_fit_pack/seed_expansion/seed.aoa.architecture-fit-pack.v0.md`

## Ownership posture

Read later plantings under these repo-home boundaries:

- `aoa-routing` owns task-to-mode hints, not sovereignty over architecture
- `aoa-agents` owns role, writer, and validator posture
- `aoa-playbooks` owns preflight composition and bounded workflow meaning
- `abyss-stack` owns runtime coordination budgets and substrate-side caps
- `aoa-evals` owns proof that an architecture mode is worth using
- `aoa-memo` owns memory decay and writeback discipline
- `Dionysus` may hold protocol addenda, but it must not become the runtime center

This pack stages architecture-fit follow-on seeds for those owners.
It does not move their meaning into `Dionysus`.

## What this pack is for

Preserve the later bounded seed set in one inspectable place:

- `AOA-SEED-R6` Architecture-Fit Routing
- `AOA-SEED-R7` Canonical Write Path Discipline
- `AOA-SEED-R8` Centralized Preflight and Validator Bottleneck
- `AOA-SEED-R9` Coordination Budget and Fanout Caps
- `AOA-SEED-R10` Architecture-Fit Eval Bundle
- `AOA-SEED-R11` Coordination Trace Decay

## What this pack is not for

- no `tenth_wave.manifest.json`
- no new `seed_index` entries
- no replacement of `navigation.next_live_seed`
- no direct placement into `seed_expansion/`
- no runtime-mode ideology masquerading as AoA doctrine
- no multi-writer canon

This pack is planted well when:

- architecture mode becomes explicit instead of folkloric
- writer authority stays bounded and inspectable
- centralized fanout is allowed only behind validator bottlenecks
- proof exists before widening coordination style
- coordination chatter remains weaker than memory canon

## Final rule

Use this prep pack to preserve architecture-fit choices until one later planting
needs them.
Do not let an architecture donor become a new sovereign layer inside the AoA / ToS ecosystem.
