# Seed Note: Questbook Boundary-Runtime Pack

## Purpose

Prepare the boundary/runtime questbook rollout from
`archive/seed_pack_exports/questbook_second_wave_seed.zip` as a named prep pack
for the derived layer, the ToS source-first boundary, and the infra runtime
layer.

This pack keeps the later rollout bounded and explicit after the source/proof
questbook contour is already in place.

## Why this belongs in Dionysus

`Dionysus` is the seed garden and dispatch layer.
This prep pack belongs here because:

- the work spans `aoa-kag`, `Tree-of-Sophia`, and `abyss-stack`
- these repos need stronger boundary posture than the earlier AoA source layers
- `abyss-stack` needs path adaptation to its real `Configs/...` structure before any later planting starts

## Bundle contents to preserve

The source artifact is `archive/seed_pack_exports/questbook_second_wave_seed.zip`.

It carries:

- `questbook_second_wave_seed/README.md`
- `questbook_second_wave_seed/APPLY_ORDER.md`
- `questbook_second_wave_seed/SEED_MANIFEST.yaml`
- `questbook_second_wave_seed/TOUCHPOINTS_NOT_INCLUDED.md`
- `questbook_second_wave_seed/COVERAGE_MAP.md`
- repo-local questbook starter surfaces for `aoa-kag`, `Tree-of-Sophia`, and
  `abyss-stack`

## Ownership posture

Read the later plantings under these repo-home boundaries:

- `aoa-kag` owns derived export and bridge obligations, not source meaning
- `Tree-of-Sophia` owns source-first knowledge architecture and must not flatten authored meaning into backlog
- `abyss-stack` owns infra and runtime guardrails and should use its existing `Configs/` root, not an invented top-level docs layout

This pack stages later bounded landings for those owners.
It does not permit identity collapse between AoA, ToS, KAG, and infra.

## Scope exclusions

This pack explicitly excludes:

- `ATM10-Agent`, which is outside this rollout line
- `aoa-sdk`, which should wait for a later separate seed
- `Dionysus` and `8Dionysus`, which should only reflect already planted upstream reality

## Suggested planting order

Keep the default order flexible but bias toward:

1. `aoa-kag`
2. `Tree-of-Sophia`
3. `abyss-stack`

## Adaptation note

The source zip carries `abyss-stack` files as if they lived at repo root.
Later planting must adapt those surfaces to `Configs/QUESTBOOK.md`,
`Configs/docs/...`, `Configs/schemas/...`, `Configs/generated/...`, and
`Configs/quests/...` so the repo stays aligned with its real ownership layout.

## Final rule

Keep this pack later, bounded, and explicit.
Do not let questbook rollout cross into derived, ToS, or infra layers until the
source/proof contour is already stable.
